# Contributing Guidelines

Thank you for your interest in contributing to this project.

## License Notice

By contributing, you agree that your contributions will be licensed under the **RK-SAL-1.0 (Robin Kutzner Source-Available License)**.

You also confirm that:

- You have the right to submit the contribution.
- Your contribution does not violate any third-party rights.

## What you can contribute

Examples of accepted contributions:

- Bug fixes
- Documentation improvements
- Code improvements
- Refactoring
- Tests

## What not to contribute

Please do not submit:

- Malicious code
- Obfuscated code without explanation
- Code that violates the project license

## How to contribute

1. Fork the repository
2. Create a new branch: git checkout -b my-change
3. Commit your changes
4. Open a Pull Request

## Attribution

All contributions remain attributed to their respective authors, but are licensed under RK-SAL-1.0.

## Right to reject

The maintainer reserves the right to accept or reject contributions for any reason.
